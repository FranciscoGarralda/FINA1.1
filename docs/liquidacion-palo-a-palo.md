# Liquidación “palo a palo” (cómo se ingresa)

1. **Transferencia (con cliente):** dos patas (salida + ingreso), cada una con cuenta, divisa, formato y **monto acordado**; no es obligatorio cotización del sistema ni del día.
2. **Pago CC cruzado:** monto y divisa del **pago real** y monto/divisa de la **cancelación en CC**; si las divisas son distintas, los montos pueden diferir; si es la misma divisa, deben coincidir.
3. **Transferencia entre cuentas:** solo **una divisa** y un solo monto entre cuentas; para dos divisas entre cuentas propias usá **Transferencia** con cliente según tu operativa.
4. Al guardar, **REAL**, **PENDIENTE** y **CC** se aplican con las validaciones ya existentes (sin FX automático impuesto).
5. El flujo antiguo entrega+cobros **exige misma moneda** en cobros; para cruzar monedas usá siempre **Transferencia** con patas explícitas.
